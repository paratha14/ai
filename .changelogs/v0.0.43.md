# v0.0.43

## Changes

- [x] Added: SSE keepalive heartbeat (`keepaliveMs`) to prevent idle connection timeouts through Cloudflare HTTP/3 QUIC, nginx, and other reverse proxies
- [x] Added: `X-Accel-Buffering: no` header to Express and Fastify SSE adapters for nginx/Cloudflare proxy compatibility
- [x] Added: `SubscriberSSEStreamOptions` interface on the H3 adapter for configuring keepalive interval
- [x] Added: `allowed_resources` support on OpenAI `mcpTool()` helper
- [x] Added: `pubsub-server.contract.test.ts` — unit contracts for keepalive behavior, disable flag, and Cloudflare-safe SSE headers across Express/Fastify adapters
- [x] Changed: H3 pubsub example docs now show manual SSE header setup with `setHeader` (required since `sendStream` does not set them)
- [x] Fixed: Long-running pipeline stages (e.g. image generation) no longer cause proxy-level connection drops due to idle timeouts

## Breaking Changes

### H3 subscriber stream requires manual SSE headers

`h3.createSubscriberSSEStream` no longer sets response headers implicitly. You must set SSE headers yourself before calling `sendStream`. This was already required in v0.0.42 when the API changed from `streamSubscriber` to `createSubscriberSSEStream`, but the docs now make it explicit.

```ts
// Before (headers were assumed to be handled)
return sendStream(event, h3.createSubscriberSSEStream(streamId, adapter));

// After (set headers explicitly)
import { sendStream, setHeader } from 'h3';

setHeader(event, 'Content-Type', 'text/event-stream');
setHeader(event, 'Cache-Control', 'no-cache');
setHeader(event, 'Connection', 'keep-alive');
setHeader(event, 'X-Accel-Buffering', 'no');

return sendStream(event, h3.createSubscriberSSEStream(streamId, adapter));
```

### `keepaliveMs` option added (non-breaking default)

The new `keepaliveMs` option defaults to `5000` (5 seconds). SSE keepalive comments (`:keepalive\n\n`) are automatically sent and ignored by all spec-compliant clients. Set to `0` to disable.

```ts
// Default behavior (5s keepalive, no change needed)
h3.createSubscriberSSEStream(streamId, adapter);

// Custom interval
h3.createSubscriberSSEStream(streamId, adapter, { keepaliveMs: 10_000 });

// Disable keepalive
h3.createSubscriberSSEStream(streamId, adapter, { keepaliveMs: 0 });
```
