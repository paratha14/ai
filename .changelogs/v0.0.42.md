## Breaking Changes

### `h3.streamSubscriber` renamed to `h3.createSubscriberSSEStream`

The H3 pub-sub adapter no longer writes directly to the Node.js response object. It returns a `ReadableStream<Uint8Array>` that works with H3's `sendStream`, which correctly handles chunked encoding across all deployment targets (Cloudflare, Nitro, reverse proxies).

```ts
// Before
import { h3 } from '@providerprotocol/ai/middleware/pubsub/server';

export default defineEventHandler(async (event) => {
  return h3.streamSubscriber(streamId, adapter, event);
});

// After
import { sendStream } from 'h3';
import { h3 } from '@providerprotocol/ai/middleware/pubsub/server';

export default defineEventHandler(async (event) => {
  return sendStream(event, h3.createSubscriberSSEStream(streamId, adapter));
});
```
