# v0.0.41

## Testing Philosophy Overhaul

v0.0.41 replaces the flat, file-per-concern test layout with a **contract-first, risk-tiered** architecture. This is the largest housekeeping release in the project's history: **~30,800 lines deleted** across 89 test files, reorganized into a structured system that maps every test to a concrete failure mode instead of inflating test counts.

### Why

The previous test suite grew organically — one file per provider, one file per feature, unit tests that mirrored internal implementation details rather than protocol contracts. This created:

- Brittle tests coupled to implementation internals rather than observable behavior
- Redundant coverage across unit and live tests with no clear gating strategy
- No distinction between "run on every PR" and "run before release"
- Test helpers scattered with no shared vocabulary for mocking providers

### What Changed

**Old structure** (flat):
```
tests/unit/core/llm.test.ts
tests/unit/providers/reasoning.test.ts
tests/unit/http/fetch.test.ts
tests/live/anthropic.live.test.ts
tests/live/openai.completions.live.test.ts
tests/live/image.live.test.ts
...
```

**New structure** (risk-tiered):
```
tests/unit/contracts/core/llm-core.contract.test.ts
tests/unit/contracts/http/retry-strategy.contract.test.ts
tests/unit/contracts/providers/provider-matrix.contract.test.ts
tests/live/canary/anthropic.live.test.ts
tests/live/nightly/anthropic.live.test.ts
tests/live/release/reasoning.live.test.ts
...
```

### Risk Tiers

| Tier | Path | Purpose | Gate |
|------|------|---------|------|
| **Contract** | `tests/unit/contracts/**` | Protocol behavior, no network. Retries, middleware lifecycle, tool loops, proxy parsing, provider wiring. | PR |
| **Canary** | `tests/live/canary/**` | Per-provider `generate` + `stream` smoke checks. Broken auth, endpoint drift, stream wiring. | Frequent |
| **Nightly** | `tests/live/nightly/**` | Structured output, tool execution, normalized errors, embeddings, caching, multi-API modes. | Scheduled |
| **Release** | `tests/live/release/**` | Expensive paths: image generation, media input, built-in tools, reasoning. Provider-scoped suites. | Manual (`RUN_RELEASE_LIVE=1`) |

### New Test Infrastructure

- **`tests/helpers/mock-llm-provider.ts`** — Deterministic mock provider using `createProvider()` directly. Supports configurable `onComplete`/`onStream` callbacks with per-attempt control, replacing ad-hoc mocking scattered across unit tests.
- **`tests/helpers/live.ts`** — Shared live test utilities: `envModel()` for model env resolution with fallback, `collectTextStream()` for stream consumption, `createAddTool()` for deterministic tool testing.
- **`tests/TESTING_STRATEGY.md`** — Living document defining scope, provider coverage matrix, acknowledged gaps, execution commands, and authoring rules.

### Authoring Rules

- One provider per live test file
- Unit tests assert protocol contracts, not implementation details
- Each live test maps to a specific regression risk
- Reuse helpers; avoid setup duplication
- Deterministic assertions; no brittle exact-text checks

## Persistence Middleware Fix

The `onStart` hook in `persistenceMiddleware` previously used a simple Set-based check to prepend missing thread messages. This could produce incorrect message ordering when the caller supplied messages that overlapped with the persisted thread.

The new implementation:

1. Builds a `Map` of request messages keyed by ID
2. Iterates thread messages in order, preferring request-supplied versions when IDs overlap
3. Appends any request messages not present in the thread
4. Replaces the request message array in-place via `splice`

This ensures thread message ordering is always respected while caller-supplied versions of duplicate messages take precedence.
