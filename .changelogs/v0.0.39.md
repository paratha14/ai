# v0.0.39

## Changes

- [x] Fixed: Pubsub middleware now emits final Turn data to subscribers, enabling compatibility with proxy provider
- [x] Fixed: Orphan stream cleanup when `streamId` is used with `.generate()` instead of `.stream()`
- [x] Fixed: H3 proxy adapter `sendError()` now properly sets HTTP status code
- [x] Added: `setFinalData()` method to `PubSubAdapter` interface for storing final Turn data
- [x] Added: `FinalDataCallback` type and optional `onFinalData` parameter in `subscribe()`
- [x] Added: Warning when pubsub middleware is misconfigured with non-streaming usage
- [x] Added: No-input support for `generate()` and `stream()` - system-prompt-only inference
- [x] Added: Zod schema support for `structure` and `tools[].parameters` - pass Zod schemas directly, auto-converts to JSON Schema
- [x] Changed: `onStreamEnd` no longer finalizes stream - `onTurn` now handles finalization for streaming
- [x] Docs: Updated h3 proxy adapter examples to use `createSSEStream` with `sendStream` for better H3/Nuxt integration

## Zod Schema Support

The `structure` parameter and `tools[].parameters` now accept Zod schemas directly. Supports both Zod v3 and v4.

```ts
import { llm, anthropic } from '@providerprotocol/ai';
import { z } from 'zod';

// Structured output with Zod
const model = llm({
  model: anthropic('claude-sonnet-4-20250514'),
  structure: z.object({
    name: z.string(),
    age: z.number(),
    tags: z.array(z.string()),
  }),
});

const turn = await model.generate('Extract user info from: John Doe, 30 years old, likes coding');
console.log(turn.data); // { name: "John Doe", age: 30, tags: ["coding"] }

// Tools with Zod parameters
const toolModel = llm({
  model: anthropic('claude-sonnet-4-20250514'),
  tools: [{
    name: 'get_weather',
    description: 'Get weather for a location',
    parameters: z.object({
      location: z.string().describe('City name'),
      units: z.enum(['celsius', 'fahrenheit']).optional(),
    }),
    run: async ({ location }) => fetchWeather(location),
  }],
});
```

### Requirements

Zod schemas must be object schemas (`z.object()`). Non-object schemas like `z.string()` or `z.array()` will throw an error:

```ts
// ✅ Valid - object schema
structure: z.object({ name: z.string() })

// ❌ Invalid - throws error
structure: z.string()  // Error: Structure schema must be an object schema
```

### Peer Dependencies

Zod is an optional peer dependency - install only if using Zod schemas:

```bash
bun add zod                    # v4+ for native JSON Schema conversion
bun add zod zod-to-json-schema # v3 requires additional package
```

| Type | Description |

### Utilities

New exports from `@providerprotocol/ai/utils`:

| Function | Description |
|----------|-------------|
| `isZodSchema(value)` | Type guard for any Zod schema (v3 or v4) |

## Breaking Changes

### PubSubAdapter interface updated

Custom adapter implementations must add `setFinalData()` method and handle `onFinalData` callback in `subscribe()`.

```ts
// Before
interface PubSubAdapter {
  subscribe(
    streamId: string,
    onEvent: SubscriptionCallback,
    onComplete: CompletionCallback
  ): Unsubscribe;
  // ...
}

// After
interface PubSubAdapter {
  subscribe(
    streamId: string,
    onEvent: SubscriptionCallback,
    onComplete: CompletionCallback,
    onFinalData?: FinalDataCallback  // NEW: optional callback for Turn data
  ): Unsubscribe;

  setFinalData(streamId: string, data: unknown): void;  // NEW: required method
  // ...
}
```

### Middleware hook timing changed

`onStreamEnd` no longer removes the stream from the adapter. Finalization now occurs in `onTurn` hook for streaming scenarios.

```ts
// Before: onStreamEnd removed stream immediately
// After: onStreamEnd waits for appends, onTurn sets final data and removes

// This change is internal - no action needed unless you were relying on
// stream removal timing in onStreamEnd
```

## Migration Notes

If you have a custom `PubSubAdapter` implementation:

1. Add `setFinalData(streamId: string, data: unknown): void` method
2. Update `subscribe()` to accept optional 4th parameter `onFinalData?: FinalDataCallback`
3. In `remove()`, call `onFinalData(finalData)` before `onComplete()` if final data was set
