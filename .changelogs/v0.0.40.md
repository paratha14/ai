# v0.0.40

## Breaking Changes

### Retry strategies refactored from classes to factory functions

Retry strategies now use the factory pattern to ensure complete isolation between concurrent requests. This prevents state sharing bugs where one request's retry state could affect another.

```ts
// Before
import { ExponentialBackoff, LinearBackoff, NoRetry, TokenBucket, RetryAfterStrategy } from '@providerprotocol/ai/http';

const instance = llm({
  model: openai('gpt-4o'),
  config: {
    retryStrategy: new ExponentialBackoff({ maxAttempts: 3 }),
  },
});

// After
import { exponentialBackoff, linearBackoff, noRetry, retryAfterStrategy } from '@providerprotocol/ai';

const instance = llm({
  model: openai('gpt-4o'),
  config: {
    retryStrategy: exponentialBackoff({ maxAttempts: 3 }),
  },
});
```

| Before | After |
|--------|-------|
| `new ExponentialBackoff({})` | `exponentialBackoff({})` |
| `new LinearBackoff({})` | `linearBackoff({})` |
| `new NoRetry()` | `noRetry()` |
| `new RetryAfterStrategy({})` | `retryAfterStrategy({})` |
| `new TokenBucket({})` | **Removed** |

### TokenBucket removed

`TokenBucket` has been removed. It conflated rate limiting (global, shared state) with retry logic (per-request, isolated state), causing bugs when the same instance was used across concurrent requests.

### Key strategies refactored from classes to factory functions

Key strategies now use the same factory pattern as retry strategies. Unlike retry factories, key strategies return `KeyStrategy` directly (no per-request isolation needed — rotation state is intentionally shared across requests).

```ts
// Before
import { RoundRobinKeys, WeightedKeys, DynamicKey } from '@providerprotocol/ai/http';

const instance = llm({
  model: openai('gpt-4o'),
  config: {
    apiKey: new RoundRobinKeys(['sk-1', 'sk-2', 'sk-3']),
  },
});

// After
import { roundRobinKeys, weightedKeys, dynamicKey } from '@providerprotocol/ai';

const instance = llm({
  model: openai('gpt-4o'),
  config: {
    apiKey: roundRobinKeys(['sk-1', 'sk-2', 'sk-3']),
  },
});
```

| Before | After |
|--------|-------|
| `new RoundRobinKeys([...])` | `roundRobinKeys([...])` |
| `new WeightedKeys([...])` | `weightedKeys([...])` |
| `new DynamicKey(fn)` | `dynamicKey(fn)` |

## Pipeline Middleware

New middleware for running async post-turn processing stages (image generation, embeddings, slug creation, etc.) with progress events streamed to connected clients.

```ts
import { llm } from '@providerprotocol/ai';
import { anthropic } from '@providerprotocol/ai/anthropic';
import { pubsubMiddleware, memoryAdapter } from '@providerprotocol/ai/middleware/pubsub';
import { pipelineMiddleware, isPipelineStageEvent } from '@providerprotocol/ai/middleware/pipeline';

const adapter = memoryAdapter();

const model = llm({
  model: anthropic('claude-sonnet-4-20250514'),
  structure: BlogPostSchema,
  middleware: [
    pubsubMiddleware({ adapter, streamId: postId }),
    pipelineMiddleware<BlogPost>({
      stages: [
        {
          type: 'slug',
          run: (turn, emit) => {
            const slug = turn.data!.title.toLowerCase().replace(/\s+/g, '-');
            (turn as { slug?: string }).slug = slug;
            emit({ slug });
          },
        },
        {
          type: 'embedding',
          run: async (turn, emit) => {
            await vectorize(turn.data!);
            emit({ embedded: true });
          },
        },
      ],
      parallel: false,        // Run stages sequentially (default)
      continueOnError: false, // Stop on first error (default)
      onStageError: ({ stage, error }) => {
        console.error(`Stage ${stage.type} failed:`, error);
      },
    }),
  ],
});

// Stages run after streaming completes, events visible to pubsub subscribers
model.stream(prompt).then(turn => {
  const extended = turn as typeof turn & { slug?: string };
  console.log(extended.slug); // Available via turn mutation
});
```

| Export | Type | Description |
|--------|------|-------------|
| `pipelineMiddleware(config)` | Function | Creates pipeline middleware instance |
| `pipelineStageEvent(stage, payload)` | Function | Creates a pipeline stage event |

### Middleware Order

Place `pipelineMiddleware` **after** `pubsubMiddleware` in the array:

```ts
middleware: [
  pubsubMiddleware({ ... }),  // Setup runs first in onStart
  pipelineMiddleware({ ... }),    // Events run first in onTurn (reverse order)
]
```

This ensures pubsub sets up before pipeline stages execute, and pipeline events emit before pubsub cleanup.

## MiddlewareContext.emit()

New method allowing any middleware to emit stream events during any lifecycle hook.

```ts
const myMiddleware: Middleware = {
  name: 'my-middleware',
  async onTurn(turn, ctx) {
    // Emit custom events that flow through onStreamEvent for all middleware
    ctx.emit({
      type: 'my_custom_event',
      index: 0,
      delta: { data: 'hello' },
    });
  },
};
```

Events emitted via `ctx.emit()` flow through `onStreamEvent` for all middleware, enabling pubsub and other event-aware middleware to process them.

## Extensible StreamEvent Types

`StreamEvent.type` now accepts custom string types while preserving autocomplete for known types:

```ts
// Before
type: StreamEventType;

// After
type: StreamEventType | (string & Record<never, never>);
```

This pattern allows middleware to define custom event types (like `'pipeline_stage'`) without requiring spec changes.

## Behavior Changes

### ctx.emit() works in all lifecycle hooks

Events emitted via `ctx.emit()` are now delivered to direct stream consumers (`for await` loops) in all lifecycle hooks:

**During streaming phase** (`onStart`, `onRequest`, `onStreamEvent`, tool hooks):
- Events are queued and yielded after each iteration
- Enables mid-stream progress/signaling from middleware

**During post-stream phase** (`onStreamEnd`, `onResponse`, `onTurn`, `onEnd`):
- Events are streamed in real-time via async channel
- Enables long-running pipeline stages to provide live progress updates

```ts
for await (const event of model.stream(prompt)) {
  if (event.type === 'pipeline_stage') {
    // Arrives in real-time as pipeline stages emit progress!
    console.log(event.delta.stage, event.delta.payload);
  }
}
```

### runTurnHook continues on error

The `runTurnHook` function now executes all middleware even if one throws:

```ts
// With middleware: [pubsubMiddleware, pipelineMiddleware]
// If pipeline stage throws, pubsub's onTurn still runs for cleanup
// The first error is re-thrown after all middleware complete
```

This ensures cleanup middleware (like pubsub) always runs, preventing subscriber hangs and adapter entry leaks when pipeline stages fail.
